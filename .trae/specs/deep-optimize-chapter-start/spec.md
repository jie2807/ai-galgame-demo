# 开启篇章模块深入优化研究 Spec

## Why
「开启篇章」模块是首页的核心交互入口，点击后进入章节任务页面（Chapter Mission Modal/Page）。当前章节页面虽然实现了基本的章节切换和开始任务功能，但在内容丰富度、交互体验、视觉表现等方面仍有优化空间。需要对模块的各个组成部分进行深入研究和优化方案设计。

## What Changes
- 章节页面整体视觉设计优化
- 章节选择和切换机制优化
- 任务开始流程优化
- 章节介绍信息展示优化
- 锁定章节的展示与解锁机制
- 自定义剧本功能整合
- 与游戏页面（/game）的衔接

## Impact
- Affected specs: 首页布局、透明化改造、章节任务系统
- Affected code: 
  - `pages/home/home.css`（chapter-start 相关样式）
  - `index.html`（chapter-start HTML 结构）
  - `pages/chapter-mission/ChapterMissionPage.js`（章节页面逻辑）
  - `pages/chapter-mission/chapter-mission.css`（章节页面样式）
  - `engine/ScriptEngine.js`（游戏引擎）

## 当前功能架构分析

### 首页「开启篇章」按钮
- 翻开书本样式的透明按钮
- 包含羽毛图标、标题「開啟篇章」、副标题、组织信息
- 点击后跳转至章节任务页面

### 章节任务页面功能
1. **章节介绍区域**（左侧 1/3）
   - 章节编号（CHAPTER I/II/III/...）
   - 章节标题（自動手記人偶、自定义剧本、???）
   - 分割线装饰
   - 章节描述文字
   - 章节集数信息（3 EPISODES）

2. **章节选择器**（底部左侧）
   - 5 个章节节点（壹/貳/叄/肆/伍）
   - 已解锁：第一章（自動手記人偶）、第二章（自定义剧本）
   - 未解锁：第三、四、五章（显示为 ???）
   - 点击已解锁章节可切换

3. **开始按钮**（底部中间）
   - 显示「開始」文字
   - 点击后跳转至游戏页面（/game）
   - 触发 ScriptEngine 初始化对应章节

4. **返回按钮**（左上角）
   - 返回首页（/home）

5. **提示消息区域**（底部）
   - 显示「任務即將開始...」等提示信息

### 游戏引擎（ScriptEngine）
- 管理章节数据（chapters 数组）
- 每个章节包含：id、number、title、desc、episodes、locked、npcs、openingScene、playerCharacter
- 初始化流程：设置章节 → 设置 NPC → 设置世界 → 添加开场消息
- 自定义剧本章节支持自定义角色、自定义世界设定、自定义 Live2D 模型

## 研究维度

### 维度 1：章节页面视觉设计
- 当前章节页面的整体布局合理性
- 左侧介绍区域的排版效果
- 底部章节选择器的视觉表现
- 开始按钮的样式和位置
- 背景视频与内容层的融合效果

### 维度 2：章节选择和切换机制
- 当前 5 个章节节点的展示方式（底部水平排列）
- 是否需要改为更直观的章节卡片或列表
- 锁定章节的展示是否需要解锁条件提示
- 章节切换时的过渡动画

### 维度 3：任务开始流程
- 点击「開始」按钮后的初始化流程
- 是否需要添加加载动画或过渡页面
- 章节数据的加载和初始化
- 与游戏页面（/game）的衔接体验

### 维度 4：章节介绍信息展示
- 当前章节编号、标题、描述的排版
- 是否需要添加章节封面图或插图
- 是否需要添加章节进度显示
- 集数信息的展示方式

### 维度 5：锁定章节与解锁机制
- 第三、四、五章显示为「???」
- 是否需要添加解锁条件说明
- 是否需要添加解锁动画效果
- 章节之间的进度关联

### 维度 6：自定义剧本功能整合
- 第二章标记为「自定义剧本」并带有编辑图标
- 自定义剧本的创建和管理流程
- 自定义角色、世界设定的配置
- 与预设章节的区分展示

### 维度 7：与首页视觉一致性
- 透明化后的视觉层次是否统一
- 颜色方案（白色文字）是否一致
- 边框和发光效果是否统一
- 按钮交互效果是否统一
