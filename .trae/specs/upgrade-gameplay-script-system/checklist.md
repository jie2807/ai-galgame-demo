# 剧本游玩功能改造升级 - 检查清单

## UI重构检查
- [ ] NPC头像下方显示好感度条，宽度与头像相近（约40px）
- [ ] 好感度条根据好感度等级显示不同颜色（初识-灰/熟悉-绿/信赖-蓝/挚友-金/羁绊-紫）
- [ ] 顶部NPC bar不再显示好感度条和NPC称号
- [ ] 顶部状态栏显示四项信息：📍地址 | ☀️天气 | 🕐时间 | 💪玩家健康
- [ ] 四项状态信息水平排列，用分隔符分隔
- [ ] UI在不同屏幕尺寸下响应式适配正常

## 剧本导入导出检查
- [ ] 导出按钮在自定义剧本编辑页面可见且可点击
- [ ] 导入按钮在首页或菜单可见且可点击
- [ ] 导出生成的JSON文件包含所有必需字段（version、metadata、worldSettings、characters、scriptConfig）
- [ ] JSON文件格式符合spec中定义的规范
- [ ] 导出的文件名格式为：{剧本标题}_{日期}.json
- [ ] 导入时验证JSON格式和必需字段
- [ ] 导入成功时显示成功提示，剧本加载到chapter 2
- [ ] 导入失败时显示具体错误信息（缺少字段/格式错误/版本不兼容）
- [ ] 导入的剧本可以正常开始游玩

## 对话流程修复检查
- [ ] 消息恢复时，NPC状态栏显示正确
- [ ] 消息恢复时，好感度条显示正确
- [ ] 消息恢复时，世界状态（地址/天气/时间/健康）恢复正确
- [ ] NPC切换后，系统消息显示"切换对话对象：{NPC名称}"
- [ ] NPC切换后，AI回复使用新NPC的身份和性格
- [ ] NPC切换后，Live2D模型更新为新NPC模型
- [ ] 流式响应中断后，isStreaming标志重置为false
- [ ] 流式响应中断后，输入框恢复可编辑状态
- [ ] 动作标签解析失败时，文本仍然正常显示
- [ ] 地点变更检测正确触发，世界状态更新
- [ ] 好感度在AI回复后正确计算和保存
- [ ] 网络错误/API错误时显示友好提示

## Live2D集成检查

- [ ] Live2D模型加载成功后显示模型
