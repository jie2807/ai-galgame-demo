# 自定义剧本双模式改造 - 检查清单

## 模式选择弹窗

- [x] 点击首页「自定义剧本」后弹出模式选择弹窗
- [x] 弹窗包含「游玩模式」与「纯文本模式」两个可点击选项
- [x] 每个选项显示模式名称、描述文案与横/竖屏提示
- [x] 点击遮罩或返回按钮可关闭弹窗
- [x] 选择模式后正确记录 `selectedCustomMode`，并进入剧本列表/编辑器

## 游玩模式横屏约束

- [x] 进入游玩模式时调用 `screen.orientation.lock('landscape')`
- [x] 不支持 orientation lock 的设备不会崩溃
- [x] 设备处于 portrait 时显示「请旋转设备」提示
- [x] 旋转到 landscape 后提示自动隐藏
- [x] 退出游戏或切换到纯文本模式时解锁方向（如支持）
- [x] 桌面端保持横屏优先布局，无强制锁定

## 纯文本模式 UI

- [x] 纯文本模式下 `gamePage` 带有 `game-mode-text` 类名
- [x] Live2D 立绘、背景层、底部 Galgame 对话框被隐藏
- [x] `#dialogueMessages` 以全屏滚动聊天记录形式显示
- [x] 消息气泡显示发送者名称、头像（NPC）或标识（玩家/旁白）
- [x] 快速回复按钮在文本模式下可见且可点击
- [x] 文本模式在竖屏与横屏下均能正常显示
- [x] 文本模式下不加载 Live2D 模型与背景图

## 自定义剧本状态隔离

- [x] 自定义章节使用 `custom_<scriptId>` 而非固定 `id=2`
- [x] 自定义 NPC 注册不覆盖官方 `gameCharacters`
- [x] 自定义世界设定、Lore、地点、物品、事件不会残留到官方剧本
- [x] 连续游玩多个自定义剧本时彼此数据不混淆
- [x] 移除 `sessionStorage.custom_game_config` 的依赖

## 启动流程一致性

- [x] 剧本列表卡片的「开始游玩」使用当前选择的模式
- [x] 编辑器「开始游玩」使用当前选择的模式
- [x] 未选择模式时默认进入游玩模式
- [x] 两种模式共用同一套 `prepareCustomScript` 初始化逻辑
- [x] `customMode` 标志被正确写入游戏状态
