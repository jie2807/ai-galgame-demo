### 关键发现

1. **上一轮清理后 `package.json` 与 `node_modules` 严重不同步**：`package.json` 只声明了 3 个 devDependencies，但 `node_modules` 里仍残留 ~100 个包、108.52 MB。这是上一轮"删 package.json"误判后的历史残留（实际 package.json 仍存在）。

2. **运行时完全不依赖 node_modules**：
   - `index.html` 通过 `<script src="...">` 加载的全部是根目录的独立库文件（pixi.min.js、live2d.min.js、live2dcubismcore.min.js、live2d-display.min.js），以及 CDN 加载的 lottie-web；
   - `index.html` 中无任何 `import` / `require` 语句；
   - 根目录的 `action-tag-parser.js`、`live2d-renderer.js`、`sprite-renderer.js` 均为纯 JS 实现，不引用任何 npm 包；
   - `scripts/`、`pages/`、`engine/` 目录下所有 JS/HTML 文件均无 node_modules 引用。

3. **无构建流程**：项目根目录无 `vite.config.js / .ts / .mjs`，`.vite` 缓存目录为空。

4. **playwright 未被实际使用**：`__verify_game_layout_screenshots/` 目录为空，说明验证脚本未产出过截图；项目代码（根目录 + scripts + pages + engine）无任何 `playwright` / `@playwright` 引用，引用全部在 node_modules 包内部相互引用。

5. **`.gitignore` 已排除 `node_modules/`**：开发者预期它可重新生成，删除不影响版本库。

## 三、三种处理方案

### 方案 A：整体删除 node_modules（最激进）

- **操作**：直接删除整个 `node_modules` 目录
- **释放空间**：约 108.52 MB
- **影响**：
  - 项目运行时（index.html + 根目录 JS）完全不受影响；
  - `package.json` 中声明的 acorn/acorn-walk/postcss 也随之消失，但项目代码未实际使用这三个包，仅是声明；
  - 如未来需要使用 acorn/postcss 等工具，可执行 `npm install` 重新安装。
- **风险**：低。需要时可通过 `npm install` 一键还原声明的依赖。

### 方案 B：仅保留 package.json 声明的依赖，删除其余（推荐）

- **操作**：删除 `node_modules` 中除 `acorn`、`acorn-walk`、`postcss`、`nanoid`、`picocolors`、`source-map-js` 之外的所有包（含 `.bin`、`.vite`、`@esbuild`、`@pixi`、`@pixiv`、`@rollup`、`@types`、`@puppeteer` 等）
- **释放空间**：约 107 MB
- **影响**：
  - 项目运行时不受影响；
  - 保留 `package.json` 与 `node_modules` 的声明一致性；
  - 后续可直接 `npm install` 而不会出现"声明的依赖找不到"。
- **风险**：极低。最稳妥方案，保留 package.json 的可复现性。

### 方案 C：保持现状（不推荐）

- **操作**：不删除任何 node_modules 内容
- **释放空间**：0 MB
- **理由**：
  - 唯一可接受的理由是"未来可能需要重新启用 vite 构建流程或 playwright 验证"；
  - 但目前项目无构建配置、验证截图为空，这些包纯属冗余。
- **风险**：高。108 MB 的死代码会让后续维护者误以为项目依赖这些包，造成混乱；同时增大仓库克隆/备份体积（虽 .gitignore 已排除，但本地占用仍在）。

## 四、推荐方案及风险说明

### 推荐方案：方案 B（仅保留 package.json 声明的依赖）

#### 推荐理由

1. **保留声明一致性**：使 `node_modules` 与 `package.json` 声明完全匹配，符合 npm 项目的常规预期；
2. **释放空间最大**：约 107 MB，与方案 A 相差仅 1 MB；
3. **可复现性最强**：后续 `npm install` 可正确还原声明的依赖；
4. **风险最低**：所有删除的包均无实际引用（运行时 / 构建时 / 测试时均未使用）。

#### 残留风险与缓解

| 风险点 | 严重度 | 缓解措施 |
|--------|:------:|---------|
| 未来若需 vite 构建，需重新安装 vite/esbuild/rollup | 低 | 体积小，`npm i -D vite` 可快速还原 |
| 未来若需 playwright 验证，需重新安装 | 低 | `npm i -D playwright` 即可，且验证截图本就为空 |
| 误删了某个隐藏引用 | 极低 | 已检查 index.html / 根 JS / scripts / pages / engine 全部无引用；`.gitignore` 已排除 node_modules，可从备份恢复 |

#### 执行清单（待人工确认后执行，本次评估不实际删除）

- [ ] 删除 `node_modules` 下除 `acorn`、`acorn-walk`、`postcss`、`nanoid`、`picocolors`、`source-map-js` 外的所有目录
- [ ] 删除 `node_modules/.bin`、`node_modules/.vite`、`node_modules/.package-lock.json`（如存在）
