# 竞品特色功能补全 Spec

## Why
经过对 SillyTavern、AI Dungeon、NovelAI、Character.AI、星野、猫箱、筑梦岛、Replika、EVE 等 9 大竞品的深度调研，发现我们项目仍缺少多项行业标配和独创功能。这些缺失功能直接影响用户的核心交互体验（消息无法编辑/重试）、AI 上下文质量（无 Persona、无 Author's Note）、沉浸感（无角色表情、无场景 BGM）和可玩性（无输入模式切换、无群聊）。需要系统性地补齐这些差距。

## What Changes
- **ADDED** 消息交互增强：消息编辑、删除、重试（Swipe 重生成）、消息分支
- **ADDED** 玩家人设系统（Persona）：定义玩家角色的姓名、外貌、性格、背景
- **ADDED** 快速回复（Quick Replies）：AI 回复后显示预设回复选项按钮
- **ADDED** 输入模式切换：Say（说话）/ Do（行动）/ Story（叙述）/ See（观察）四种模式
- **ADDED** 作者备注（Author's Note）：可自定义的 prompt 注入，支持深度和频率控制
- **ADDED** 角色表情系统：根据对话情绪动态切换角色表情/立绘
- **ADDED** Lorebook 高级功能：正则关键词、深度插入位置、递归扫描、概率触发、包含组、选择性逻辑
- **ADDED** NPC 关系网：NPC 之间的好感度和互动关系
- **ADDED** 场景 BGM 系统：根据场景/地点/时间自动切换背景音乐
- **ADDED** UI 主题自定义：主题色、背景图、聊天样式可配置
- **ADDED** 日记/编年史系统：自动记录关键事件，可回顾
- **ADDED** 成就系统：里程碑追踪和展示
- **ADDED** 提示管理器（Prompt Manager）：可视化 prompt 构建和调试

## Impact
- 受影响能力：对话交互、AI 上下文构建、角色系统、Lorebook、UI 系统、存档系统
- 受影响代码：
  - `index.html` 内嵌 `SimpleChatEngine`（消息处理、prompt 构建）
  - `index.html` 游戏输入区 UI
  - `index.html` Lorebook 系统
  - `index.html` 游戏状态管理
  - `index.html` CSS 样式

## ADDED Requirements

### Requirement: 消息交互增强
系统 SHALL 提供消息级别的交互操作，包括编辑、删除、重试和分支。

#### Scenario: 编辑消息
- **WHEN** 用户点击自己或 AI 的某条消息的编辑按钮
- **THEN** 消息内容变为可编辑状态
- **WHEN** 用户修改内容并确认
- **THEN** 该消息更新为新内容
- **THEN** 该消息之后的所有消息被标记为"已废弃"（可选择保留或删除）
- **THEN** AI 基于新消息重新生成回复

#### Scenario: 删除消息
- **WHEN** 用户点击某条消息的删除按钮
- **THEN** 该消息从对话中移除
- **THEN** 对话历史更新，后续消息不受影响

#### Scenario: 重试（Swipe）
- **WHEN** 用户点击 AI 消息的重试按钮
- **THEN** 系统重新调用 AI 生成新回复
- **THEN** 用户可以左右滑动切换不同版本的回复
- **THEN** 每条 AI 消息最多保留 5 个版本

#### Scenario: 消息分支
- **WHEN** 用户从某条消息编辑后选择"创建分支"
- **THEN** 原消息链保持不变
- **THEN** 从编辑点创建新的消息分支
- **THEN** 用户可以在分支间切换

### Requirement: 玩家人设系统（Persona）
系统 SHALL 允许用户定义自己的角色身份，AI 在对话中感知和使用这些信息。

#### Scenario: 创建人设
- **WHEN** 用户打开人设编辑面板
- **THEN** 可以填写：姓名、性别、年龄、外貌描述、性格特征、背景故事
- **THEN** 人设信息注入到 System Prompt 的玩家角色部分

#### Scenario: 切换人设
- **WHEN** 用户创建多个人设
- **THEN** 可以在不同人设之间快速切换
- **THEN** 切换人设后，AI 使用新人设信息

#### Scenario: 人设持久化
- **THEN** 人设数据保存到 localStorage
- **THEN** 页面刷新后人设不丢失

### Requirement: 快速回复（Quick Replies）
系统 SHALL 在 AI 回复后显示预设的快速回复选项。

#### Scenario: 显示快速回复
- **WHEN** AI 生成回复后
- **THEN** 在消息下方显示 2-4 个快速回复按钮
- **THEN** 按钮内容基于 AI 回复和当前场景动态生成

#### Scenario: 自定义快速回复
- **WHEN** 剧本作者在编辑器中定义快速回复模板
- **THEN** 在游玩时按模板显示快速回复选项
- **THEN** 支持条件触发（特定场景/好感度/地点显示不同选项）

#### Scenario: 点击快速回复
- **WHEN** 用户点击某个快速回复按钮
- **THEN** 该文本作为用户消息发送
- **THEN** 快速回复按钮消失

### Requirement: 输入模式切换
系统 SHALL 支持四种输入模式，影响 AI 对用户输入的理解和回复方式。

#### Scenario: 切换输入模式
- **WHEN** 用户选择不同的输入模式
- **THEN** 输入框样式和占位符文本相应变化：
  - **Say（说话）**：输入框显示「说些什么...」，AI 将用户输入视为角色对话
  - **Do（行动）**：输入框显示「做什么...」，AI 将用户输入视为角色行动
  - **Story（叙述）**：输入框显示「叙述故事...」，AI 将用户输入视为旁白/叙述
  - **See（观察）**：输入框显示「观察什么...」，AI 生成环境描述

#### Scenario: 模式影响 prompt
- **THEN** 不同模式在用户消息前添加不同的前缀指令
- **THEN** AI 根据模式调整回复风格

### Requirement: 作者备注（Author's Note）
系统 SHALL 允许用户在对话中注入自定义指令，控制 AI 的行为方向。

#### Scenario: 设置作者备注
- **WHEN** 用户在作者备注输入框中写入内容
- **THEN** 该内容按设定频率注入到 prompt 中
- **THEN** 注入深度可配置（0=最靠近 AI 回复，越大越远）

#### Scenario: 插入频率
- **THEN** 可设置每 N 条消息插入一次（1=每次，2=每隔一次，0=不插入）
- **THEN** 默认频率为 1，深度为 4

### Requirement: 角色表情系统
系统 SHALL 根据对话情绪动态切换角色表情/立绘。

#### Scenario: 情绪检测
- **WHEN** AI 生成回复后
- **THEN** 系统分析 AI 回复中的情绪关键词
- **THEN** 匹配到对应的表情类型（开心/悲伤/愤怒/惊讶/害羞/平静）

#### Scenario: 表情切换
- **WHEN** 检测到情绪变化
- **THEN** 角色立绘/头像切换为对应表情版本
- **THEN** 表情切换有过渡动画

#### Scenario: 自定义表情映射
- **WHEN** 剧本作者定义情绪关键词到表情的映射
- **THEN** 优先使用自定义映射
- **THEN** 未定义时使用默认映射

### Requirement: Lorebook 高级功能
系统 SHALL 升级现有 Lorebook 系统，增加正则关键词、深度插入位置、递归扫描、概率触发、包含组和选择性逻辑。

#### Scenario: 正则关键词
- **WHEN** Lorebook 条目的关键词是有效的正则表达式
- **THEN** 使用正则匹配而非纯文本匹配
- **THEN** 支持大小写敏感/不敏感配置

#### Scenario: 深度插入位置
- **WHEN** Lorebook 条目设置了插入位置
- **THEN** 支持：角色定义前、角色定义后、示例消息前/后、指定深度（@D）、作者备注顶/底部
- **THEN** 插入位置影响条目对 AI 输出的影响力

#### Scenario: 递归扫描
- **WHEN** 一个 Lorebook 条目的内容触发了另一个条目的关键词
- **THEN** 被触发的条目也被激活和注入
- **THEN** 递归深度限制为 5 层，防止无限循环

#### Scenario: 概率触发
- **WHEN** Lorebook 条目设置了触发概率（0-100%）
- **THEN** 即使关键词匹配，也按概率决定是否注入
- **THEN** 可用于创建随机事件

#### Scenario: 包含组
- **WHEN** 多个条目属于同一包含组
- **THEN** 同组中只有权重最高的一个条目被注入
- **THEN** 可用于创建"多选一"的随机描述

#### Scenario: 选择性逻辑
- **WHEN** Lorebook 条目设置了次要关键词和选择性逻辑
- **THEN** 支持 AND ANY（任一匹配）、AND ALL（全部匹配）、NOT ANY（任一不匹配）、NOT ALL（全部不匹配）
- **THEN** 次要关键词作为主要关键词的额外过滤条件

### Requirement: NPC 关系网
系统 SHALL 支持 NPC 之间的好感度和互动关系。

#### Scenario: 定义 NPC 关系
- **WHEN** 剧本作者在编辑器中定义 NPC 间关系
- **THEN** 可以设置：NPC A 对 NPC B 的好感度、关系类型（朋友/敌人/恋人/师徒/陌生人）
- **THEN** 关系信息注入到 AI prompt 中

#### Scenario: 关系影响对话
- **WHEN** 两个有关系的 NPC 同时在场
- **THEN** AI 根据关系类型调整 NPC 之间的互动方式
- **THEN** 好感度变化时关系可能升级或降级

### Requirement: 场景 BGM 系统
系统 SHALL 根据场景/地点/时间自动切换背景音乐。

#### Scenario: BGM 自动切换
- **WHEN** 游戏地点或时间发生变化
- **THEN** 系统根据地点类型和时间段选择匹配的 BGM
- **THEN** BGM 切换有淡入淡出效果

#### Scenario: 手动控制
- **WHEN** 用户点击 BGM 控制按钮
- **THEN** 可以暂停/播放/切换 BGM
- **THEN** 可以调节音量

#### Scenario: 自定义 BGM
- **WHEN** 剧本作者上传或指定 BGM 文件
- **THEN** 按场景/地点/时间配置 BGM 映射
- **THEN** 使用 Web Audio API 播放

### Requirement: UI 主题自定义
系统 SHALL 支持界面主题、背景、聊天样式的自定义。

#### Scenario: 切换主题
- **WHEN** 用户打开主题设置面板
- **THEN** 可选择预设主题（暗金/暗蓝/暗紫/浅色/樱花）
- **THEN** 可自定义主色调、背景图、字体大小
- **THEN** 主题设置保存到 localStorage

#### Scenario: 聊天样式
- **THEN** 可选择气泡样式/小说样式/终端样式
- **THEN** 可调整消息间距、头像显示方式

### Requirement: 日记/编年史系统
系统 SHALL 自动记录游戏中的关键事件，形成可回顾的编年史。

#### Scenario: 自动记录
- **WHEN** 发生关键事件（地点变更、NPC 加入/离开、重要物品获得、好感度变化、战斗结果）
- **THEN** 系统自动在编年史中添加条目
- **THEN** 条目包含：时间戳、事件类型、事件描述

#### Scenario: 查看编年史
- **WHEN** 用户打开编年史面板
- **THEN** 按时间线显示所有记录的事件
- **THEN** 可按事件类型筛选
- **THEN** 可搜索关键词

### Requirement: 成就系统
系统 SHALL 提供成就追踪和展示功能。

#### Scenario: 成就触发
- **WHEN** 玩家达成预设条件（首次对话、好感度满级、收集特定物品、到达特定地点等）
- **THEN** 弹出成就解锁通知
- **THEN** 成就记录到玩家档案

#### Scenario: 成就面板
- **WHEN** 用户打开成就面板
- **THEN** 显示所有成就（已解锁/未解锁）
- **THEN** 未解锁成就显示达成条件提示

### Requirement: 提示管理器（Prompt Manager）
系统 SHALL 提供可视化的 prompt 构建和调试界面。

#### Scenario: 查看 prompt 结构
- **WHEN** 用户打开提示管理器
- **THEN** 显示当前 prompt 的完整结构（系统指令/世界设定/角色定义/Lorebook/作者备注/对话历史）
- **THEN** 每个部分显示 token 占用估算

#### Scenario: 调整 prompt 顺序
- **WHEN** 用户拖拽调整 prompt 部分的顺序
- **THEN** 下次生成时使用新顺序

#### Scenario: 预览完整 prompt
- **WHEN** 用户点击"预览"
- **THEN** 显示即将发送给 AI 的完整 prompt 文本
- **THEN** 高亮显示各部分来源
