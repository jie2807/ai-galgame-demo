# 舆论锻造所B站结构重构与Agent系统 Spec

## Why
当前UI存在严重设计错误：1) 头像下拉菜单内容错误（应显示私信/个人空间等B站同款功能，而非投稿管理/数据中心等创作中心功能）；2) 头像下拉位置错误（出现在右下角而非头像正下方）；3) 首页分类标签无法切换；4) 搜索功能不工作；5) 私信和通知未集成到头像区域；6) 首页视频排布布局未对标B站。此外，系统缺乏核心玩法：需要创建Agent UP主自动发布视频、搭建事件系统驱动内容生成、实现视频文字动画。需要按B站真实结构重新设计UI布局，并构建完整的Agent驱动内容系统。

## What Changes
- **BREAKING**: 重构头像下拉菜单——严格对标B站真实设计，包含B站所有同款功能入口
- **BREAKING**: 修复头像下拉位置——确保面板出现在头像按钮正下方
- **BREAKING**: 首页视频排布布局全面对标B站真实首页
- 修复首页分类标签切换功能（推荐/热门/最新/动画/知识/科技）
- 修复搜索功能
- 将私信和通知集成到头像下拉菜单中
- 新增Agent UP主系统：1-2个拥有不同观点的AI UP主，仅在高消耗模式下启用
- 新增事件系统：基于玩家刷新和操作生成背景事件，驱动Agent UP主制作带观点的视频
- 新增视频文字动画：视频发布时生成HTML文字动画预览
- 所有生成类功能（AI视频/故事/标题）均需加入进度条显示
- 参考AI视频生成手法，实现简化版视频内容生成

## Impact
- Affected specs: `refine-bilibili-ui-immersion`（头像下拉内容需重构）
- Affected code:
  - `index.html`：头像下拉HTML重构、个人空间页面增加创作中心功能Tab、首页布局重构、标签交互修复、搜索修复、Agent系统HTML、进度条组件
  - `pages/creator-wars/creator-wars.css`：头像下拉位置修复、首页B站布局样式、进度条样式、个人空间创作中心样式
  - `CreatorWarsEngine` JavaScript：Agent UP主AI逻辑、事件系统、视频文字动画生成、高/低消耗模式切换

1. **5列网格布局**：每行5个视频卡片，响应式调整
2. **卡片结构**：缩略图(16:9) + 标题(两行截断) + UP主名·播放量

### Requirement: Agent UP主系统（高消耗模式）
系统 SHALL 创建1-2个拥有不同观点的AI UP主Agent，**仅在玩家开启高消耗模式时启用**，低消耗模式下不启动Agent。

**高/低消耗模式**：
- 在设置面板中新增"性能模式"选项
- **低消耗模式（默认）**：不启动Agent UP主，不生成事件，不生成AI视频内容，首页仅显示玩家自己发布的视频
- **高消耗模式**：启动Agent UP主系统，自动生成事件和视频，首页显示玩家+Agent的视频

**Agent UP主设计**：

**Agent 1: "真相调查员" 🔍**
- 观点倾向：追求真相、质疑权威、深度分析
- 视频风格：调查类、深度分析类
- 对事件的态度：倾向于挖掘事件背后的真相，质疑官方说法

**Agent 2: "热点追踪者" 📡**
- 观点倾向：追逐热点、情绪化表达、大众视角
- 视频风格：热点评论、情绪输出类
- 对事件的态度：倾向于跟随大众情绪，放大热点效应

**Agent行为逻辑**：
1. 事件系统生成新事件后，Agent根据自身观点倾向生成视频标题和简介
2. Agent的视频自动发布到首页信息流
3. Agent的视频带有其独特的观点标签
4. Agent的视频会获得随机的播放量、评论数等数据
5. Agent之间可能对同一事件发布对立观点的视频

**Agent数据结构**：
```javascript
{
    name: '真相调查员',
    avatar: '🔍',
    level: 5,
    fans: 23000,
    viewpoint: 'investigative',
    videos: [],
    publishedVideos: []
}
```

#### Scenario: 高消耗模式下Agent自动发布视频
- **WHEN** 高消耗模式开启且事件系统生成新事件
- **THEN** 每个Agent根据自身观点倾向自动生成并发布一条视频

#### Scenario: 低消耗模式下Agent不启动
- **WHEN** 低消耗模式开启
- **THEN** Agent UP主不活动，首页仅显示玩家自己的视频

### Requirement: 事件系统
系统 SHALL 搭建基于玩家操作和定时刷新的事件系统，生成背景故事供Agent UP主制作视频。

**事件类型**：
- 🏢 企业丑闻（某公司被曝数据造假/环境污染/劳工问题）
- 📢 政策变动（新法规/监管措施/行业标准变更）
- 🔬 科技突破（AI新进展/新能源/医疗突破）
- 🌍 社会事件（公共安全/教育公平/社区治理）
- 🎭 娱乐热点（明星争议/文化现象/网络流行）

**事件生成逻辑**：
- 玩家每次进入首页时，有概率生成新事件（仅高消耗模式）
- 事件包含：标题、描述、影响领域、热度值、持续时间
- 事件影响舆论指标（好感度、信任度等）
- Agent UP主根据事件自动生成视频

**预设事件模板**（至少10个）：
1. 企业数据泄露事件
2. 新能源政策发布
3. AI伦理争议
4. 网红虚假宣传曝光
5. 食品安全问题
6. 教育改革方案
7. 社交平台算法争议
8. 环保组织报告
9. 医疗事故调查
10. 文化遗产保护争议

#### Scenario: 事件触发
- **WHEN** 高消耗模式下玩家刷新首页
- **THEN** 有概率生成新事件，Agent UP主自动发布相关视频

### Requirement: 生成类功能进度条
所有生成类功能（AI视频生成、故事生成、标题生成等）SHALL 显示进度条，让玩家实时查看进度。

**进度条设计**：
```
┌──────────────────────────────────────────────────────────┐
│  🤖 AI内容生成中...                                       │
│  ┌────────────────────────────────────────────────────┐  │
│  │████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│  │
│  └────────────────────────────────────────────────────┘  │
│  正在生成视频标题... 65%                                   │
└──────────────────────────────────────────────────────────┘
```

**需要进度条的生成环节**：
1. **事件生成**：显示"正在生成新事件..."进度
2. **标题生成**：显示"正在生成视频标题..."进度
3. **故事/简介生成**：显示"正在生成视频内容..."进度
4. **关键帧生成**：显示"正在生成视频帧..."进度
5. **视频发布**：显示"正在发布视频..."进度

**进度条样式**：
- B站品牌蓝色渐变（`linear-gradient(90deg, #00a1d6, #00c4e8)`）
- 圆角，高度6px
- 进度百分比文字显示在右侧
- 生成阶段文字显示在进度条上方
- 整体卡片式容器，白色背景，圆角阴影

**实现方式**：
- 由于不调用外部API，实际生成是即时的
- 使用 `setTimeout`/`requestAnimationFrame` 模拟渐进进度
- 每个阶段持续1-2秒，总过程5-8秒
- 进度条平滑过渡，不会突然跳变

#### Scenario: Agent生成视频时显示进度
- **WHEN** Agent开始生成视频
- **THEN** 显示进度条，依次显示各生成阶段，进度从0%到100%

#### Scenario: 玩家发布视频时显示进度
- **WHEN** 玩家点击"发布"按钮
- **THEN** 显示生成进度条，完成后显示成功提示

### Requirement: 视频文字动画
视频发布时 SHALL 生成简单的HTML文字动画预览，模拟视频内容。

**文字动画设计**：
- 使用HTML/CSS动画实现
- 视频卡片缩略图区域显示文字动画而非静态图片
- 动画内容：视频标题文字逐字显示 + 关键词闪烁 + 背景渐变
- 动画时长：3-5秒循环

**4种动画风格**：
- 调查类：深色背景+白色文字逐字出现
- 热点类：红色背景+黄色文字弹跳出现
- 知识类：蓝色背景+白色文字淡入
- 科技类：黑色背景+绿色代码风格文字

#### Scenario: 视频文字动画显示
- **WHEN** 玩家浏览首页或视频详情页
- **THEN** 视频缩略图区域显示文字动画，而非空白或静态图片

### Requirement: 简化版AI视频内容生成
系统 SHALL 实现简化版的AI视频内容生成，参考AI视频生成的思路。

**实现方式**：
- 不调用外部AI API（保持离线可用）
- 使用预设模板 + 事件数据 + Agent观点 自动组合生成视频内容
- 视频内容包含：标题、简介、关键帧文字、标签、分类
- "视频播放"时展示HTML动画序列（多帧文字+过渡效果）

**视频内容生成流程**：
1. 事件触发 → Agent选择事件
2. Agent根据观点生成标题和简介（显示进度条）
3. 系统从模板库选择匹配的动画风格（显示进度条）
4. 生成关键帧文字（3-5帧，每帧一句核心观点，显示进度条）
5. 组装为完整的视频对象（显示进度条）

**视频对象结构**：
