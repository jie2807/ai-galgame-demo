# 随说：千界 —— 评审答辩 Q&A

## 1. 项目的技术架构是怎样的？前端、AI、存储如何协作？

- 项目是一个纯前端 PWA 单页应用（`index.html` + `styles.css` + 多页面 CSS 模块 + 内联 JS），无需后端即可运行，便于快速部署到 GitHub Pages / Vercel / Netlify。
- 核心模块包括：首页与章节选择、Galgame 式对话页、自定义剧本编辑器（CCE）、Creator Wars 社交模拟、抽卡/角色/书架/阅读器。
- AI 交互通过浏览器 `fetch` 直接调用 OpenAI-compatible 接口（文本 / 生图 / 视频 / TTS），支持官方 Agnes 模型以及用户自定义的 OpenAI、DeepSeek、通义千问、Ollama 等服务商。
- 状态与存档使用 `localStorage` / `sessionStorage`；API Key 默认只存 `sessionStorage`，关闭浏览器即清除，降低泄露风险。
- Live2D 使用 PixiJS + Cubism SDK 本地渲染；TTS 支持 Edge TTS（免费）、Fish Audio、火山、讯飞以及本地 Supertonic（WebGPU 离线）。

## 2. 项目使用哪些 AI 模型？API 成本如何控制？

- 文本叙事默认使用官方 Agnes 免费模型，也兼容 gpt-4o-mini、deepseek-chat、qwen-turbo 等低成本模型；玩家可自行切换。
- 生图、视频、TTS 均为可选模块，默认关闭或提供免费路径（Edge TTS、Supertonic 本地 TTS），避免强制消耗高成本 API。
- 成本策略：对话接口按需调用、支持上下文长度与字数配置；自定义剧本编辑器可一键生成完整世界观与事件，减少反复调用。
- 未来可通过用户自定义 API Key 的方式转嫁模型成本，官方只维护轻量代理或免费额度，适合个人/小团队运营。

## 3. 用户数据安全和隐私如何保障？

- 项目默认纯前端运行，不收集服务器端用户数据；对话历史、存档、自定义剧本全部保存在浏览器本地存储。
- API Key 采用 `sessionStorage` 优先策略，关闭标签页/浏览器后自动清除；只有在用户主动选择“保存到本地”时才会写入 `localStorage`。
- 支持“导出全部数据”和“导入全部数据”，用户可完全掌控自己的存档与剧本资产。
- 若使用官方 Agnes 接口，请求通过 HTTPS 传输；若用户担心 Key 暴露，可部署 Cloudflare Workers / Vercel Serverless 做反向代理，将 Key 藏在服务端。

## 4. 移动端和 PWA 适配做了哪些工作？

- 项目已完成横屏锁定、等比缩放、触屏交互修复、480px/360px 断点适配，并新增 `manifest.webmanifest`、Apple touch icon、theme-color。
- 针对移动端补齐了长按删除、消息编辑/删除入口、剪贴板回退、固定底栏键盘避让、`inputmode` 与 `font-size≥16px`、全局 tap-highlight 抑制等细节。
- Live2D Canvas、阅读器翻页区、抖音视频流等区域设置了 `touch-action`，避免手势冲突；模态窗口使用 `overscroll-behavior: contain` 防止背景连锁滚动。
- PWA 支持“添加到主屏”，启动时为全屏/横屏模式，接近原生应用体验。

## 5. 与 AI Dungeon、Character.AI、传统视觉小说相比，你们的差异化在哪里？

- **相比 AI Dungeon**：我们提供官方完整剧本（末世方舟 / 修仙穿越）与结构化事件系统（时间线/阈值/随机/终局事件），不是完全开放的沙盒，降低玩家“不知道做什么”的迷茫。
- **相比 Character.AI**：我们不只做单轮角色聊天，而是把角色嵌入带世界状态、好感度、物品、多结局的连续叙事中，并提供 Creator Wars 社交经营玩法。
- **相比传统视觉小说**：剧情由 AI 实时生成，玩家选择真正影响世界状态与结局；同时保留 Galgame 的立绘、BGM、打字机效果等沉浸元素。
- 核心差异在于“结构化剧本 + AI 即兴叙事 + 社交模拟”三者结合，既有序章、世界观，又有无限延展性。

## 6. 项目的商业化路径是什么？

- **短期**：作为免费 Web/PWA 体验产品，通过 TapTap、社交媒体、创意大赛获取种子用户，验证留存与付费意愿。
- **中期**：推出“创作会员”，解锁更多自定义剧本槽位、高级 AI 模型、生图/视频额度、专属角色与皮肤；创作者可分享/出售自己的剧本。
- **长期**：搭建 UGC 剧本市场，官方与创作者分成；将成功剧本 IP 化，拓展有声书、短剧、周边等衍生形态。
- 成本端采用“用户自带 Key + 官方代理”双模式，既降低初期服务器压力，也保留了增值服务的空间。

## 7. 未来一年的产品路线图是怎样的？

- **Q1（参赛-测试期）**：完成名称与 IP 风险整改，上线 PWA 可玩版本，收集核心玩家反馈，优化官方剧本节奏。
- **Q2（UGC 编辑器）**：升级自定义剧本编辑器，支持导入小说/剧本、AI 辅助续写、多人协作与世界树分支可视化。
- **Q3（社交与内容）**：完善 Creator Wars 的订单、排行、AI 评论系统；开放剧本分享与轻量社区。
- **Q4（IP 与多平台）**：引入 AI 生成视频片头/短剧，探索微信小程序/快应用版本，推进原创角色 IP 联名。

## 8. 项目原名“紫罗兰永恒花园”，现在改为“随说：千界”，版权/IP 风险如何处理？

- 原名称与动漫《Violet Evergarden》（中译《紫罗兰永恒花园》）高度重合，角色 VIOLET、C.H 邮政公司、自动手记人偶等设定也接近该 IP，存在较大版权投诉与上架审核风险。
- 已确定原创名称“随说：千界”，寓意“随心而说、千万世界”，既保留书信/叙事意象，又摆脱具体 IP 绑定。
- 替换范围包括：`manifest.webmanifest` 的 name/short_name、首页 `<title>`、应用图标与品牌文案、角色设定中的 VIOLET 相关元素、C.H 邮政公司等高风险设定。
- 后续所有官方素材（海报、Logo、角色立绘）将围绕“随说：千界”重新设计，确保参赛与商业化的合规性。

## 9. AI 生成内容的可控性如何保证？会不会出现脱离世界观或“胡说”？

- 我们通过四层机制约束 AI：世界观词条库（`gameWorldInfo`）、剧本主控文档（`playerDesc` 四段式）、状态变量与事件系统、情绪关键词与动作标签解析。
- 每次请求都会携带当前世界状态、NPC 关系、地点、天气、历史对话摘要，确保输出贴合上下文。
- 自定义剧本编辑器提供基础/中等/高级三档生成提示词，越高级的世界观约束越详细，降低“幻觉”。
- 玩家可手动编辑任何 AI 生成的剧情、消息和角色设定，保留人对叙事的最终控制权。

## 10. 项目部署和演示链接如何准备？有没有考虑国内访问？

- 主方案使用 GitHub Pages 部署仓库静态文件，无需服务器审核，5 秒内可加载首页。
- 备用方案同步部署到 Vercel / Netlify / Cloudflare Pages，防止 GitHub Pages 在国内访问受限。
- 若需要临时给评审演示，可使用 ngrok 或 Cloudflare Tunnel 暴露本地服务，并明确告知链接为临时性质。
- API Key 安全方面，公开部署版本默认使用官方 Agnes 免费接口；高阶模型建议用户自行配置 Key，或通过服务端代理隐藏 Key。
