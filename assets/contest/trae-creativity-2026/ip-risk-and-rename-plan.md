# IP 风险评估与更名计划

## 1. 原名称的 IP 风险

项目原名为 **「紫罗兰永恒花园」**，存在较高的版权与商标风险：

1. **与日本动漫 IP 中文译名完全一致**：「紫罗兰永恒花园」是日本动画 / 轻小说作品《Violet Evergarden》的官方中文译名，该 IP 由京都动画等版权方持有，具有较高知名度和商业价值。
2. **核心设定高度重合**：项目早期资料中出现了「C.H 邮政公司」「自动手记人偶」「VIOLET 角色」「紫罗兰/书信」等元素，均与《Violet Evergarden》的核心设定直接对应。
3. **上架与参赛风险**：
   - 应用商店（TapTap、App Store、各安卓商店）审核可能因版权争议拒绝上架。
   - 版权方投诉可能导致下架、赔偿或法律纠纷。
   - 在 Trae AI 创造力大赛等公开赛事中，使用侵权名称会显著影响评审印象，甚至导致资格问题。

**风险等级：高。**

## 2. 更名决策

经评估，项目正式更名为 **「随说：千界」**（英文名 SuiShuo: Thousand Realms）。

### 更名理由

- **原创性**：「随说：千界」为完全原创名称，与任何已知动漫、游戏、小说 IP 无直接冲突。
- **契合产品形态**：「随说」强调自然语言驱动的低门槛创作，「千界」呼应 AI 生成的多世界、多分支、多结局体验。
- **延展性强**：不绑定特定题材（如末世、修仙、西幻等），未来可自由扩展世界观。
- **中文语感好**：四字结构简洁、易记，适合作为参赛名称与上架名称。

## 3. 最小替换范围（仅规划，不实际执行）

为完成从「紫罗兰永恒花园」到「随说：千界」的切换，至少需要替换或检查以下内容：

### 3.1 应用元信息

- `manifest.webmanifest`：
  - `name`：由「紫罗兰永恒花园」改为「随说：千界」
  - `short_name`：由「紫罗兰花园」改为「随说」
  - `description`：同步更新为新的项目描述
- `index.html`：
  - `<title>` 由「紫罗兰永恒花园 - 首页」改为「随说：千界 - 首页」
  - `apple-mobile-web-app-title` 由「紫罗兰花园」改为「随说」
  - 首页标题、Logo 文字、副标题中的旧名称
  - 任何 `<meta name="description">` 中的旧名称

### 3.2 角色与设定文案

- 检查 `data/characters.js`、`data/game-world-info.js`、`data/game-chapters.js` 等数据文件，移除或替换以下关键词：
  - VIOLET / 薇尔莉特 / 紫罗兰（如作为角色名出现）
  - C.H 邮政公司 / 自动手记人偶 / 自动手记人偶（Auto Memory Doll）
  - 与原 IP 强绑定的地名、组织名、道具名
- 当前官方剧本已切换为「末世方舟」与「修仙」题材，但仍需全局搜索确认无残留。

### 3.3 图片与视觉资产

- 检查并替换以下明显关联原 IP 的视觉素材：
  - `images/violet-avatar.jpg`
  - `images/violet-avatar-lg.jpg`
  - `images/evergarden-flowers.jpg`
  - `models/sprite/violet-sprite.js` 及其引用
  - 任何带有紫罗兰花卉、书信、C.H 邮政公司徽章风格的图标/封面
- 替换为原创主视觉：如羽毛笔、星界门、书卷、千界之门等意象。

### 3.4 样式与主题（名称层面）

- 检查 `styles/variables.css` 及其他 CSS 文件中的注释或类名是否包含旧名称或角色名。
- 颜色主题（暗紫、暖金）本身不侵权，可保留，但需避免在文案中将其与「紫罗兰」直接绑定。

### 3.5 文档与规格

- `.trae/specs/prepare-taptap-store-listing-materials/project-summary.md`
- `.trae/specs/prepare-trae-creativity-contest-entry/` 相关文件
- 任何 README、参赛文案、开发者故事中的旧名称引用

## 4. 执行原则

- **本次任务只做规划文档，不实际修改代码或资源文件。**
- 正式更名应在单独的「rename-suishuo-promo」规格任务中执行，并逐条验证替换结果。
- 更名后建议进行一次全局文本扫描（搜索「紫罗兰」「Violet」「Evergarden」「VIOLET」「C.H」等关键词），确保无遗漏。
- 同时准备新的应用图标（512×512 / 192×192）和主视觉海报，替换旧 IP 相关素材。
